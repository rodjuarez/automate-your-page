Lesson 1: Introduction to "Serious" Programming

Computer Science

Computer Science is all about how to solve problems through establishing sequencial and very specific steps that a computer can follow.

Computer

Computers by themselves are incapable of doing anything. In order to benefit from their processing power a series of commands, that is, a "program", must be given to them in a language that they can read and follow, that will tell them exactly what to do and how to do it.

Computer Program

A program is a precise sequence of steps that a computer executes. Various programs can be combined and executed, or a very large one. These programs serve a purpose, whether entertainment, such as games; retrieve information, such as search engines; display information, such as Web browsers; calculate, such as spreadsheets; etc.

Programming Language

A programming language is made of rules to write commands that will tell a computer what and how to execute those commands. There are many programming languages. An example of a programming language is Python.

Python

Python is a high grade programming language. It not only runs or executes programming code, or commands, but it is also an interpreter that converts the code into instructions that the computer can understand and follow. It has an editor page where the code can be written, and a run option to execute the code.

Grammar

Just as people use languages made of rules as to how to communicate, programming languages also use specific rules to do the same with computers. However, human languages are imperfect because they are ambiguos and depend of context to be understood. Programming languages cannot be ambiguos and cannot depend on cotext or the computer would not know what to do. Computers are stupid in the sense that they lack the intelligence to figure out and solve the ambiguity and in which context the commands should be executed.

Unambiguos, specific and strict rules then must be followed in a good programming language to avoid problems with the code.

Python Expressions

A valid Python expression is a command, or statement, that is written in accordance to the preset rules for this language. When it follows these rules it is said to be a legal statement. One that fails to follow these rules it is said to be an ilegal statement.

A video was given for the better understanding of the Python grammar and the understanding of legal expresions.

Compilers

A compiler is a program that runs programs and then writes new programs. The first compiler was written by Rear Admiral Grace Hopper with the name of Cobol, that allowed for faster processing by computers.

The difference between compilers and interpreters, such as Python, is that an interpreter runs and writes new programs at the same time, thus called a high level programming language.

Lesson 2: Variables and Strings

What is Debugging?

Python, as many other programming languages, requires to be written in a very specific syntax, that is, without typos and mistakes for it to work, otherwise it leads to errors in the code.

Debugging is correcting code mistakes, called "bugs". For that a systematic system must be developed for finding and fixing the bugs in the code.

What is a variable in Python?

One of the most important tools developed to increase the speed and readability of the code is through the use of variables.

Variables are a way to assign names to values. En example would be: If speed_of_light is a variable and we assign it the value of 299,792,458 meters per second; and if we would like to know what half the speed of light is, we could use the command print to calculate the following equation: print speed_of_light / 2, which would produce 149,896,229 in meters per second.

Comments to the code could be added by using a hash tag "#" before the comment, which would not print.

It is not a rule but it is a convention not to write variables with capitalized letters.

What does it meant to assign a value to a variable? How is it done?

You assign a value to a variable by first specifying the name of the variable. The name of a variable can be anything but if the name is more than one word, the words must be joined by an underscore. For example: speed_of_light would be the correct way to specify a varialbe.

Then we would add an equal sign "=" on the right side of the varialbe. It is important to note that the sign equal does not mean the same in math than in programming, in which it does not establish equality but that the value on the right has been assigned to the variable on the left. Example: Variable: speed_of_light = 299792458. Thus, we have assigned the value 299,792,458 to the variable speed_of_light.

What are some ways variables are useful?

Variables can be useful to programmers in many ways:

  • They improve code readability by using names that allow for a more logical way for a programmer to write and later follow the code.
  • They provide an easier way to store the value of important data.
  • By reassigning new values to the same variable one can build more complex code, that gives us the ability to write more complex and faster solutions. For example: the variable time = time+1
What is the difference between 2 + 2 and "2" + "2"?

In Python, 2 is an integer while "2" is a string, or a series of characters.

The code 2 + 2 would produce the sum of these two integers, or4. while the code "2" + "2" would join both characters into a the string "22".

What is a String?

A string is a series of characters surrounded by quotes, or double quotes. Quotes and double quotes cannot by combined, we can use one or the other. However, quotes can be included within double quotes, or viceversa. When a string is outputted it prints without the quotes.

Augusta Ada King, Countess of Lovelace (1812-1852) was the first one to introduce strings to code.

What is String Concatenation?

Operations can be used with strings. The plus "+" sign using strings does not produce an addition but the union (concatenation) of characters into a string such as: "Welcome" + "back" + "!!" would produce the result Welcome back!!

The multiply operator can also be used with strings resulting in multiplying the string or strings in question. For example: The result of "love" * 3 would produce the result lovelovelove.

Extracting sub sequences from strings cannot be done with numbers, only with strings. The code convention to do this is through the sequence, followed by an expression within square brackets: string[<expression>]. The expression would output a number that is the position withnin the string where the sequence is found. The string always begins with the first position of the first character with the value of the integer 0. Thus the string "hello" would have 5 positions, where h=0, e=1, l=2, l=3, o=4.

The string that is a subsequence of a character in s, begins from position start and it ends in the position stop, not counting the last character, represented by "-1". So, in:

Through -1 it would produce the last character in the string. So print word[-1] would produce the result e.

A criterias could be used to find a subsequence coded as string[0:1] would produce the result wa, corresponding to positions 0 and 1.

The find Method

In order to extract a subsequence from a string a tool can be used that facilitates the process, this is a method with the name of find. This is written as: string.find[string]. This will find the first instance of the subsequence in the string. If the subsequence cannot be found the result would produce the result "-1".

Parameters can be added to the find method to find other than the first ocurrance of the subsequence in the string. For example, we will look for subsequences in the variable Einstein = 'Before God we are all equally wise and equally foolish'

Lesson 3: Input -> Function -> Output

What is a function or procedure?

A function its a group of instructions or procedures that indicate how and when to manipulate data. The funciton requires to define parameters or values that it stores once but that it can use multiple times.

An example could be to define a function defined as sum with two parameters or values a,b. we could specify that we want a return to be a + b. The output would produce the sum that we input for a and b.

How do we code a function and how do we use it?

The first thing to code a funtion is to define it with the command def followed by the name of the function, such as sum and the parameters we are going to use in between parenthesis, and separated by comas. A colon ends the first line.

The second line will contain the procedures of what we want to do with the input parameters. For example, we could specify that we want to add the parameters such as a + b.

The third line could include a variable or just the print command where we could input the values given to a and b. Example:

def sum(a,b):
return a + b
print sum(1+2)
>>>3

What is the difference between making and using a function?

Madking refers to the code used to specify the procedure and parameters of what the function is for. Using refers to the input and output produced by the function as we input the values we want to calculate or change and the results that the function produces.

How do functions help programmers avoid repetition?

The value of functions reside in their power to store procedures that can be reused or called over and over again without having to write them again. By changing the parameters values the same function can also be suded to calculate values that use the same procedure or function.

What happens if a function doesn't have a return statement?

The return command instructs the code to produce the type of output we desire to get fromt he code and the input paramaters we have selected., of course, the function would only execute but it would not produce any output.

When should you use a function?

A function must be defined before it can be used. A function is useful to separate a portion of code, or block of code from the rest, which makes it easier to manipulate, modify, or debug. The power of its use come to light when we realize that we will be running the same code with the same input values or different ones throughout the program, so unnecessary repetition is avoided, and thus the risk for more mistakes and longer processing. That is why functions are so valuable.

Lesson 4: Control Flow & Loops If & While

Equality decisions

Equality operands include "less than" <, "greater than" >, "equal to" ==, "not equal to" !=. It is important to note that = and == are not the same. The first denotes assignment. In other words, a value has been assigned to a variable, while the second one refers to equality between two values.

The construct is <number><operator><number>. Example: 2 < 3 returns True, while 21 < 3 would result in a False statement.

if Statement

The if statement produces a True or False comparison result. Its construct is:
if <text expression>:
<Block>

The or operand

The or operand results in evaluating an expression, if it is True (validates the first expression) the following expressions are not evaluated. If it is False, it will continue validating the second expreesion, and so on. The matrix is: True or False = True. False or True = True. True or True = True. False or False = False.

Loops

Loops is a procedure that repeats over and over the same procedure as long as it continues validating the procedure as True. Once it returns False the loop stops.

While loops

While Loops are procedures that continue looping while certain condition remains True. As soon as the condition becomes False the While Loop stops. The construct is similar to the if expression:
while<Test Expression>:
<Block>

The while loop will conitnue looping for every True statement. When it becomes False it will continue with the rest of the code.

The break statement

The break statement stops the loop even if the while statement remains True. This is done to avoid a never ceasing loop, so that the rest of the code can be called. It is used after a while loop and its construct is:
while<Test Expression>:
.....if<Break Test>: #is True
........break #stop looping continue with code

Lesson 5: Debugging

Debugging

Debugging means finding and fixing code errors that prevent the code to work, or work correctly. It could also be rewriting code to make it faster or simpler.

Debugging strategy
There are 5 debugging strategies to follow:

  1. Look up error messages when a program crashes

    The last line in the error message point out the error. From there one can go backwards to understand the problem and in which line of code it refers to. It is always useful to go online to find out more about that error code.

  2. Work using example code

    When making changes to the code it is important to keep copies of the code instead of changing it so much that one looses track of what worked. It can be left as a comment, or it can be copied to another file. It is important to work out the code by small pieces at the time, ensuring that things work as you work through it.

  3. Make sure the example code work

    Sometimes it is a good idea to look up for similar existing code. However, it is important to test the code to ensure that it is bug free and that it works well with your own coding, ensuring that it produces the results that you expect. Otherwise, you will have to modify it or not use it at all.

  4. Check chunks of code as you go printing it

    Sometimes the code doesn't crash but it doesn't produce the desired results, which is harder to fix than when it crashes because we don't obtain an indication of where to find it and what type of error it is. The only way to work aroudn this is printing statements as you go through the code until you find the problem.

  5. Keep and compare old versions

    It is also important to save the code that works before adding more to it because if all else fails you can always go back to it as a starting point. It also serves as a small library of working code in my opinion.

Lesson 6: Structured Data: Lists & For Loops

Lists

A string is a sequence of characters only. The power of lists is that they contain a sequence of anything, numbers, characters, a mix of numbers and characters, and empty list and lists whithin lists. Another big difference is that lists support mutability where as strings do not. List store complex items.

Lists construct is:
<list>[<expression>,<expression>,...]. Example:
p = [1,2,3,'a','b']

Nested lists

As said before, a list can be placed within another list. However, when printing the elements on the list, the nested list is considered as only one element.

Example: p = [1,2,[3,4]]. The restul would be 3 elements: 1, 2 and the nested list [3,4] as one element.

Mutation

Mutation is changing the value of a list once we have created it. Every time a list is created it should be with in [ ] brackets. Every element in the list is assigned a number. The first element in the list is element or position "0", the second element is "1", and so on. Once an element is changed the list is mutated. Example:

p = ["r", "o", "a", "d"]
p[0] = "l" would mutate the list to:
p = ["l", "o", "a", "d"]

Aliasing

Aliasing is when two different ways are used to address the same list. Example:

p = [1,2,3]
q = p
p = [1,2,3]
q = [1,2,3] where both "p" and "q" refer to the same list [1,2,3]. However, if changes are then done to list "p" again list "q" will remain the same.

Append

The append operand for lists is like a procedure but it is a method, so it works with strings too. It appends or incorporates an element at the end of a list. Its construct is: <list>.append(<element>). It is important to note that the append operand mutates the list only, it does not create a new list.

+ Plus

The "+" plus operand works like concatenation for strings. Its construct is:
<list> + <list>. Example:

[0,1] + [2,3] would result in [0,1,2,3]. It is important to note that "+" plus creates a new list, it does not mutate it. The difference between append and "+" is that append mutates the list but "+" creates a new list by concatenation.

Len

Len stand for length. It works like a procedure. it produces the number of elements in a llist or the length of the list. Its construct is:
len(<list>). It will not work with strings. Example:

len([0,1] would result in 2.

for Loops

for loops is an easier way to loop for loops rather than using while. The way it works is that it reads it as for each element in the list evaluate the block of code, going through the list. Its construct is:
for <name> in <list>:
....<Block>

Index

As its name implies, the index operand indexes the list, taht is, looks for an element in the list. If the element is not found in the list is returns an error message. when several similar elements are contained in a list, the index operand will only return the first element. Its construct is:
<list>.index(<value>)

The in procedure call

Sometimes a better way to find an element in a list is to use the "in" procedure. It looks for a value in a list. The result is produces is a bolean answer, True or False, so it does not produce an error message as the index procedure. Its construct is:
<value> in <list>, such as 3 in p, find the element 3 in the list p.

Another way to construct it is by using its negative:
<value> not in <list>, which is the same as to saying:
not <value> in <list>, but easier to read.

The first thing to do is not to panic because if you lose your head you won't be able to work through it and will not work in a systematic manner.

2. What are the inputs?

It is important to understand the problem and what it is asking. The first thing to understand is to understand what the inputs are. What goes in?

What are the outputs?

Solve the problem

Work thorugh some examples by hand. Start using pseudo code that will help you understand the problem and how to solve it. Then the pseudo code can be writen in correct syntax once we know how to solve the problem.

Simple mechanical solution

Simple mechanical solutions refer to pseudo code, which is a the way a human would solve the problem. Then code some examples checking if that solves the problem and produces the right output. Consider what the code does not address. It is important to add comments at every step of the way that will allow you and others to understand your code at a later date and will help with debugging.

Develop incrementally and test as you go

When writing the code one should write it by blocks, incrementally, testing each block as you go. Then see if you can find a simpler way to produce the same result. In other words, simplify the code that will make it faster and easier to understand, a simpler solution because the bigger the code, the more complex it will be, and the greater the number of mistakes it might produce.
